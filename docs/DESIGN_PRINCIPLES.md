## Purpose

This document defines the UI and UX philosophy for Market Pulse AI.

## Design Philosophy

The interface should feel professional, calm, trustworthy, and useful. It should help users understand market information quickly without hiding uncertainty.

The product should look credible, but visual polish must never reduce readability, accessibility, or transparency.

## Explain Before Recommending

The dashboard should help users understand why a score, recommendation, or warning exists before asking them to trust it.

AI views should be paired with evidence, limitations, and risk context.

## Readable Before Beautiful

The interface should prioritize legibility, contrast, spacing, and hierarchy.

A beautiful interface that is difficult to read is not acceptable for financial research.

## Simple Before Complicated

Simple workflows should be preferred over complex interactions. The user should not need to learn the product before understanding the market summary.

Progressive disclosure is preferred when advanced information is valuable but not needed immediately.

## Evidence Before Confidence

Confidence should follow evidence. The UI should not make weak evidence look strong through color, animation, or aggressive language.

## Responsive Product Standard

### Mobile-First

Mobile views must preserve core understanding, avoid overlap, and support key workflows without horizontal scrolling.

### Desktop Professional

Desktop views should support dense but organized research workflows, similar to professional market dashboards.

## Accessibility

Market Pulse AI should support:

- Keyboard navigation
- Visible focus states
- Sufficient contrast
- Clear labels
- Non-color-only status communication
- Accessible dialogs and popovers
- Readable text at common viewport sizes

## Internationalization

The product should support Thai and English with consistent meaning across languages.

Financial terminology may remain in common professional English where appropriate, but user-facing explanations should be understandable for the selected language.

## Interaction Consistency

Common controls should behave consistently:

- Info icons explain concepts
- Buttons perform clear actions
- Tabs switch views
- Accordions expand educational content
- Badges communicate status
- Unavailable states explain missing data

## Visual Complexity

Avoid unnecessary visual complexity. Color, motion, charts, and density should serve understanding.

Every visual emphasis should have meaning.
